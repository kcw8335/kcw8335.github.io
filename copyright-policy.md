# 저작권 원칙

이 사이트는 논문 원문을 보관하거나 재배포하는 곳이 아니라, 공개된 서지 정보와 원문 링크를 바탕으로 개인적인 해석과 투자 리서치 메모를 정리하는 곳입니다.

## 올리지 않는 것

| 항목 | 원칙 |
|---|---|
| 논문 PDF | 저장소와 사이트에 올리지 않습니다. SSRN, NBER, arXiv, DOI, 출판사 페이지로 링크합니다. |
| 초록 전문 | 초록을 그대로 복사하지 않습니다. 필요한 경우 한두 문장 수준으로만 짧게 인용하고 대부분은 요약합니다. |
| 표와 그림 | 논문 표, 차트, 그림, 스크린샷을 그대로 가져오지 않습니다. 필요한 경우 직접 재계산하거나 직접 만든 도표를 씁니다. |
| 유료 본문 | 페이월 안의 문장을 길게 옮기지 않습니다. 공개적으로 접근 가능한 서지 정보와 내 해석 위주로 씁니다. |
| 데이터 파일 | 라이선스가 불분명한 원자료를 저장소에 포함하지 않습니다. |

## 리뷰 작성 규칙

1. 제목, 저자, 연도, 저널, DOI/SSRN/arXiv/NBER 링크를 먼저 적습니다.
2. 논문의 표현을 베끼지 말고 내 말로 질문, 방법론, 결과, 한계를 정리합니다.
3. 직접 인용은 꼭 필요한 경우에만 씁니다.
4. 긴 문단, 표, 그림, 수식 전개를 그대로 가져오지 않습니다.
5. 투자 아이디어는 논문 결론과 분리해서 적습니다.
6. 원문 접근은 링크로 유도하고, 파일 재배포는 하지 않습니다.

## 안전한 글쓰기 형식

| 쓰기 방식 | 위험도 |
|---|---|
| 논문 링크 + 내 요약 + 내 비판 | 낮음 |
| 핵심 문장 짧은 인용 + 출처 명시 | 보통 |
| 초록/본문 긴 복사 | 높음 |
| PDF, 표, 그림, 스크린샷 업로드 | 높음 |

## 삭제 요청 대응

권리자가 문제를 제기하면 해당 부분을 우선 비공개 처리하거나 삭제하고, 원문 링크와 독자적 요약만 남깁니다. GitHub는 저작권 침해 신고에 대해 DMCA 절차를 운영하므로, 의심스러운 자료는 처음부터 저장소에 넣지 않는 것을 기본 원칙으로 합니다.

## 면책

이 사이트의 글은 학습과 리서치 목적의 개인 메모이며 투자 조언이 아닙니다. 저작권 판단은 사안별로 달라질 수 있으므로, 애매한 경우에는 원문 링크만 남기고 직접 인용이나 이미지 사용을 피합니다.
